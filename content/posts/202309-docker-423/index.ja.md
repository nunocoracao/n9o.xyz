Docker Desktop 4.23が利用可能になりました。Docker InitでのASP.NET Coreサポート、注意が必要な設定変更を警告する設定整合性チェック、クロスドメインID管理など、多数の機能強化が含まれています。このリリースではクイックサーチも改善され、コンテナ、アプリ、Docker Hub、ドキュメント、任意のボリュームを横断して検索し、クイックアクション（開始/停止/削除）を実行できます。Macユーザーにパフォーマンス向上を提供するため、VirtioFSがデフォルトに設定されました。Docker Desktop 4.23リリースでは、Macユーザーは従来のネットワーク接続を使用したネットワークパフォーマンスの向上も体験できます。

この投稿では、Docker Desktopの最新リリースの新機能と更新内容について詳しく説明します。

## ASP.NET CoreとDocker Init
ASP.NET Coreのサポート追加を発表できることを嬉しく思います。Dockerの初心者でも熟練者でも、Docker InitはASP.NET Coreプロジェクトのコンテナ化を効率化します。プロジェクトフォルダでdocker initコマンドを実行し、最新のDocker Desktopバージョンを使用するだけで、Docker Initがカスタマイズされたdockerfile、Composeファイル、.dockerignoreファイルを生成します。

ASP.NET Core固有の改善として、マルチアーキテクチャビルドのDockerFileのサポートとコメントも改善しました。この進歩により、ユーザーは異なるCPUアーキテクチャ間でイメージを共有し、Azure、AWS、GCPなどのクラウドプロバイダーへのデプロイを効率化できます。さまざまなアーキテクチャに適合するイメージを作成し、クラウドデプロイの柔軟性と効率を向上させます。

最新のDocker Desktopバージョンをインストールして始めましょう。次に、コマンドラインでプロジェクトディレクトリでdocker initを実行します。Docker Initに重労働を任せて、コアタスクである素晴らしいアプリケーションの作成に集中しましょう！

## 設定整合性チェック
新しい設定整合性チェックでDocker Desktopをスムーズに実行できます。複数のローカルアプリケーションやツールを使い続けながら、設定変更を手間なく行えます。このアップデートは設定変更を自動的に検出して警告し、シンプルなクリックでDocker Desktop設定を再確立して、中断のない開発を追跡し確保できます。

IdPで組織内のメンバーにロールを割り当てることができます。ロールを設定するには、ロールを割り当てたい人のオプションのユーザーレベル属性を使用できます。

## クイックサーチの改善
必要なときに開発者がエッセンシャルリソースにシームレスにアクセスできるよう、クイックサーチ機能が大幅にアップグレードされました。ユーザーは以下を簡単に見つけられます：

コンテナとComposeアプリ：ローカルシステム上の任意のコンテナまたはComposeアプリを簡単に特定できます。さらに、環境変数へのクイックアクセスを得て、開始、停止、削除などの基本的なアクションを実行できます。
Docker Hubイメージ：公開Docker Hubイメージ、ローカルイメージ、リモートリポジトリからのイメージなど、クイックサーチは高速で関連性の高い結果を提供します。
拡張機能：特定の拡張機能についてさらに詳しく知り、ワンクリックでインストールを効率化します。
ボリューム：ボリュームを簡単にナビゲートし、関連するコンテナについての洞察を得ます。
ドキュメント：Docker公式ドキュメントからの貴重な支援に、Docker Desktop内から直接アクセスできます。
強化されたクイックサーチは、リソースの発見と管理のための究極のツールであり、開発者に比類のない利便性を提供します。

## MacユーザーのためにVirtioFSで高性能ファイル共有を標準化
Docker Desktop 4.23は、docker run -vまたはCompose YAMLのバインドマウントを通じてコンテナとファイルを共有する際に大幅なパフォーマンス向上を提供するため、macOS 12.5以降でVirtioFSをデフォルトとして使用するようになりました。VirtioFSは、ボリュームマウントやネットワーク共有なしでコンテナがファイルにアクセスできるようにすることで、ファイル転送のオーバーヘッドを最小化します。

ネットワークファイル転送プロトコルをスキップすることで、ファイル転送も高速化されます。パフォーマンス改善を測定したところ、ファイル転送時間が7:13.21秒から1:4.4秒に短縮され、85.15%の速度向上が確認されました。測定された改善はファイルサイズ、実行中の他のアプリ、コンピューターのハードウェアによって異なることに注意してください。

## MacユーザーのためのDocker Desktopネットワーク速度改善
Docker Desktop 4.23には、Macユーザー向けのネットワークパフォーマンス改善が含まれています。コンテナが従来のネットワーク接続を必要とする場合、ユーザーは以下の方法でネットワークパフォーマンスの向上を体験できます：

公開ポートへのアクセスが約10倍高速化
伝送制御プロトコル（TCP）が約1.5〜2倍高速化
これらの恩恵を受けるためにユーザーアクションは不要です — 4.23に更新されたすべてのMacユーザーのネットワークが高速化されます！
